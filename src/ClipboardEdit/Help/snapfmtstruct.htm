File structure

The Clipboard Snapshot (.CSNAP) file format allows for storing data about what formats are in the clipboard, as well as when it was taken.

Offset Size Description
0x00 4 Magic (0x0F1B0A6D)
0x04 4 Version
0x08 4 Format count
0x0C Variable Format entries
After format entries 8 FILETIME taken

The file starts with a header (0x0F1B0A6D) denoting that it is a clipboard snapshot file. After the header is the 4-byte (32-bit) version number. The following version numbers are currently available:

  • 0x10000000 - Version 1

Following the version number is the count of clipboard formats in the snapshot, also stored as a 4-byte integer.

Afterwards, the actual array of the formats is stored. Each format follows the structure of an unsigned 4-byte integer identifying the clipboard format, such as CF_TEXT, followed by the 4-byte length, and finally the actual data. This should be repeated for the amount specified in the count field earlier.

Finally, at the end is the time and date the snapshot was taken, specified via a Windows FILETIME in the UTC time zone. This time is specified as an 8-byte (64-bit) integer of 100-nanosecond intervals since 00:00:00 UTC on January 1, 1601.
